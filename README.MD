# 表格内容确认工具 v3

一个基于 **Alpine.js + Tailwind 风格 UI** 的纯前端网页小工具，用于对多表头文本数据进行解析、在线编辑、逐行确认、统计分析，并支持多表管理及多种导出 / 导入能力。

> 本工具完全在浏览器本地运行，不依赖后端服务，适合本地文件校对、寄存器表、配置表等内容的整理与确认。

---

## 功能特性

- **多表管理**
  - 支持多个独立表格：每个表有自己的表头、行数据、统计信息和历史记录。
  - 支持在当前表追加数据（在当前表下方加行）。
  - 支持“新建表追加”，在页面下方新增一个完全独立的新表，不影响原表。

- **多种导入方式**
  - 通过文本框或拖拽文件导入表头和数据内容。
  - 支持按“格式（schema）”区分不同表头布局。
  - 支持从示例数据快速体验。

- **在线编辑与确认**
  - 双击单元格即可编辑内容。
  - 支持右键菜单：确认/撤销确认、编辑行、复制行、删除行、删除列等。
  - 行状态区分：已确认、未确认、追加行等，并有颜色与标记提示。
  - 支持添加自定义列、删除列。

- **多维度筛选与搜索**
  - 全局搜索：对当前所有可见表按任意字段模糊搜索。
  - 筛选模式：全部 / 已确认 / 未确认 / 追加行。
  - 按表头格式统计各格式的行数、确认进度等。

- **导出功能**
  - **📤 导出（当前表）**：将当前表中选定条件的行导出为一份居中、美观的 HTML 报告。
  - **📤 导出所有表格**：将所有有数据的表统一导出到一份 HTML 报告中，每个表一个分节，并展示各自统计信息。
    - 报告主标题可通过页面上的「报告标题」输入框自定义。
  - **💾 导出当前修改**：将当前表的所有修改（表头、行数据、格式、历史等）导出为 JSON 文件（version: 1）。
  - **💾 导出所有修改**：将 **所有表** 的配置与数据一次性导出为一个 JSON 文件（version: 2）。

- **导入历史修改**
  - 在初始界面和每个表操作区都提供 **“导入历史数据”** 按钮。
  - 可从之前导出的 JSON 恢复当前表的全部数据与编辑状态，方便中断后继续。

- **导出报告样式**
  - 深色主题，内容区域居中，最大宽度可读性优化（当前约 1412px）。
  - 展示导出时间、总记录数、确认率、列数、格式分布等信息。
  - 行状态使用“已确认 / 待确认”徽章清晰标记。

## 使用方式

### 1. 启动

1. 确保文件在本地存在：
   - `TableCheck.html`
   - `tailwind.css`
   - `alpine.min.js`
2. 使用浏览器（推荐最新的 Chrome / Edge）**直接打开** `TableCheck.html`。

> 工具为单页应用，无需任何后端或构建步骤。

### 2. 导入数据

1. 在页面上方的“表头输入”和“内容输入”区域填入文本，或从文本文件拖拽导入。
2. 点击解析按钮（界面中的解析操作按钮，通常在输入区域下方）生成表格。
3. 如有多种表头格式，可配置多个“格式（schema）”，工具会按格式拆分并统计。

### 3. 编辑与确认

- 双击任意单元格进入编辑模式，`Enter` 保存，`Esc` 取消。
- 使用右键菜单可对当前行或列执行：
  - 确认/撤销确认
  - 编辑行、复制行、删除行
  - 删除列
- 上方统计卡片会实时反映当前表的总记录数、已确认、未确认及整体进度。

### 4. 多表操作

- 使用“在当前表追加”将新数据追加到当前表。
- 使用“新建表追加”将新数据作为一个独立新表追加到页面底部。
  - 原表内容与统计保持不变。
  - 新表拥有独立的表头、行数据和编辑历史。

## 导出 / 导入说明

### 导出当前表 HTML 报告

- 在某个表的操作栏中点击 **📤 导出**。
- 可选择：
  - 仅导出已确认行
  - 按表头格式过滤导出
  - 是否包含自定义列
- 导出结果为单表 HTML 报告文件，可直接在浏览器中查看或分享。

### 导出所有表格 HTML 报告

- 在顶部搜索栏右侧：
  - 在「报告标题」输入框中设置主标题（默认：`表格内容确认报告（全部）`）。
  - 点击 **📤 导出所有表格**。
- 生成一份包含所有有数据表格的 HTML 报告，每个表一个章节，标题为你设置的报告标题。

### 导出修改为 JSON

- **当前表**：
  - 在表操作栏点击 **💾 导出当前修改**。
  - 生成 `tablecheck-modifications-*.json`（version: 1）。
- **所有表**：
  - 在顶部搜索栏右侧点击 **💾 导出所有修改**。
  - 生成 `tablecheck-all-modifications-*.json`（version: 2），结构示意：
    ```json
    {
      "version": 2,
      "exportedAt": "2026-03-10T12:34:56.789Z",
      "tables": [
        {
          "id": 1,
          "name": "表1",
          "headers": ["..."],
          "customColIndices": [],
          "schemas": [{ "name": "Format-A", "headers": ["..."] }],
          "rows": [
            {
              "__id": 1,
              "__displayIndex": 1,
              "__data": ["..."],
              "__custom": {},
              "__schemaIdx": 0,
              "__confirmed": true,
              "__appended": false
            }
          ],
          "idCounter": 10,
          "editHistory": [/* 最近 100 条历史 */]
        }
      ],
      "_nextTableId": 3
    }
    ```

### 导入历史修改

- 在初始界面或某个表操作栏点击 **📂 导入历史数据**，选择之前导出的 JSON 文件。
- 对于单表 version:1 的 JSON，会导入到当前激活表。
- 导入后：
  - 表头、行数据、格式信息与确认状态都会恢复。
  - `idCounter` 和部分编辑历史会一并恢复。

## 开发说明（可选）

- 本项目目前是单页、无构建的静态页面：
  - 主文件：`TableCheck.html`
  - 依赖：`tailwind.css`、`alpine.min.js`
- 主要逻辑集中在 `tableApp()` 的 Alpine.js 组件内，包括：
  - 多表结构：`tables[]`、`activeTableIndex`、`currentTable` getter。
  - 导入解析、行编辑、确认逻辑。
  - 各类导出函数：`doExport()`、`doExportAllTables()`、`exportModifications()`、`exportAllModifications()` 等。
