# 表格内容确认工具 v3

基于 **Alpine.js + Tailwind 风格 UI** 的纯前端单页工具，用于多表头文本的解析、在线编辑、逐行确认、多表管理与多种导出/导入。

> 完全在浏览器本地运行，无后端依赖，适合寄存器表、配置表等本地校对与确认。

---

## 功能概览

### 多表管理（Tab 切换）

- 多表以 **Tab** 形式切换，同一时间只显示一个表的内容。
- 每个表独立：表头、行数据、格式（schema）、统计、编辑历史互不干扰。
- **在当前表追加**：向当前 Tab 的表末尾追加行。
- **新建表追加**：新建一个独立表（新 Tab），原表不变；支持「默认表头」快速填入预设列名（Name、Bits、Permission 等）。
- **清空此表**：只清空当前表的行数据，保留表头与格式，可继续追加。
- **删除表**：Tab 右侧 × 删除整表（至少保留一个表）。
- **Tab 标题**：双击 Tab 可编辑表名；**格式名**（如 Format-A）：在表头格式统计卡片中双击可编辑。

### 导入

- **数据导入**：手动输入 / TXT 文件 / 表格样式（TSV、CSV 等），解析后生成表格。
- **导入历史数据**：选择此前导出的 JSON，恢复表数据与编辑状态；支持 version 1（单表）和 version 2（多表）。
- **撤消上一步**：导入后若选错文件，可一键恢复到本次导入前的状态。

### 编辑与确认

- 双击单元格编辑，`Enter` 保存、`Esc` 取消；右键菜单：确认/撤销确认、编辑行、复制行、删除行、删除列。
- 行状态：已确认、未确认、追加行，有颜色与徽章区分。
- 支持添加/删除自定义列，表头双击可改列名。

### 筛选与统计

- 全局搜索、筛选：全部 / 已确认 / 未确认 / 追加行。
- **所有表确认统计**：汇总表数量、总记录、已确认、未确认、总确认率（带进度条）。
- 单表与各格式（schema）的统计在各自区域展示。

### 导出

- **📤 导出（当前表）**：按条件将当前表导出为单份 HTML 报告（可勾选仅已确认、按格式过滤、是否含自定义列）。
- **📤 导出所有表格**：将所有有数据的表合并为一份 HTML，每表一节；文件名与报告标题一致。
- **💾 导出当前修改**：当前表完整状态导出为 JSON（version: 1）。
- **💾 导出所有修改**：所有表 + 报告标题与备注 导出为一份 JSON（version: 2）。

### 报告标题与备注

- 主页面 **「导出报告备注」** 模块（与数据导入同风格，默认显示）：
  - **报告标题**：导出 HTML 时用作主标题和默认文件名。
  - **备注**：多行文本，会显示在导出 HTML 的标题下方（支持换行）。
- 报告标题与备注会随 **导出/导入 JSON** 一并保存与恢复。

## 使用步骤

### 启动

- 本地需有：`TableCheck.html`、`tailwind.css`、`alpine.min.js`（后两者需与 HTML 同目录，或自行修改 HTML 中的引用路径）。
- 用浏览器直接打开 `TableCheck.html`（推荐 Chrome / Edge）。

### 导入数据

1. 在「数据导入」区域选择 手动输入 / TXT 文件 / 表格样式，填入或拖入表头与数据。
2. 点击 **🚀 解析** 生成表格；多表头格式会按 schema 拆分并统计。
3. 无数据时也可点击 **📂 导入历史数据** 从 JSON 恢复。

### 编辑与多表

- 双击单元格或使用右键菜单编辑、确认行、删行删列。
- 切换 Tab 可编辑不同表；用「在当前表追加」或「新建表追加」增加数据或新表。
- 需要时在「导出报告备注」中填写报告标题和备注，再导出 HTML 或 JSON。

## 导出 / 导入说明

### 导出 HTML

- **单表**：在对应表操作栏点 **📤 导出**，在弹窗中设置筛选条件后导出。
- **所有表**：在有数据时顶部工具栏点 **📤 导出所有表格**；报告标题与备注取自主页面「导出报告备注」，文件名与报告标题一致。

### 导出 / 导入 JSON

- **导出当前修改**：导出当前表（version: 1），含 `reportTitle`、`reportRemark`、表头、行、格式、历史等。
- **导出所有修改**：导出全部表及报告标题、备注（version: 2），含 `tables`、`_nextTableId`、`reportTitle`、`reportRemark`。
- **导入历史数据**：选择 JSON 后恢复表数据；若文件中有 `reportTitle` / `reportRemark` 会一并恢复。
- 导入后可点 **↩️ 撤消上一步** 回到导入前状态。

## 浏览器与性能

- 推荐 **Chrome / Edge**。Safari 下部分视觉效果（如 `backdrop-filter`）可能不同，功能正常。
- 页面做了预加载、非阻塞 CSS、筛选结果缓存、弹窗按需挂载、表格区域 `content-visibility` 等优化，以提升加载与滚动性能。

## 项目结构

| 文件 | 说明 |
|------|------|
| `TableCheck.html` | 主页面，包含全部 UI 与 `tableApp()` 逻辑 |
| `tailwind.css` | Tailwind 样式（需自行放入同目录或修改引用） |
| `alpine.min.js` | Alpine.js 框架（需自行放入同目录或修改引用） |
| `tablehead.txt` | 示例表头文本，可用于「数据导入」测试 |
| `tableval.txt` | 示例数据文本（如寄存器表），可与表头一起解析 |

- 逻辑集中在 `tableApp()` 中：多表 `tables[]`、`activeTableIndex`、当前表 getter、导入解析、编辑与确认、各类导出/导入及撤消。
